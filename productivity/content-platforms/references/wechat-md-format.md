# 公众号文章 MD 格式规范

**用途**：`wechat-article-daily.py` 生成的 Markdown 文件格式标准，直接发送给用户用于复制到公众号后台。

---

## 整体结构

```
# 标题（一句话概括，冲突感强）

开头段落（hook，1-2句，引出故事）

---

## 01（或故事A标题）

故事A正文，包含具体场景、人物、数字、对话
> 引用对话框（story_a_dialogue）
故事A结尾（golden1位置）

---

## 02（或故事B标题）

故事B正文
故事B结尾（golden2位置）

---

## 03（或观点）

核心观点，一句话

---

## 该怎么做？

行动建议，3-4条bullet

---

## 金句

- 金句1
- 金句2
- 金句3

---

*话题标签：#标签1 #标签2 #标签3*

---

**互动问题**（争议性，能引发转发讨论）
```

## 颜色标注规范（重要·2026-05-31更新）

**核心格式**：用户自行二次转换公众号，因此 MD 文件中统一使用 HTML `<span>` 标签标注颜色，不使用 Markdown 粗体 `**`。

### 金句高亮
```html
<span style="color: rgba(255, 69, 0, 1)">金句全文</span>
```
- 金句整体作为一个 `<span>` 包起来，不拆散（方便用户一键复制转换）
- 放在故事结尾或观点后

### 关键数字标红
- 金额/年龄/百分比等关键数字用同样的格式：
```html
<span style="color: rgba(255, 69, 0, 1)">数字</span>
```
- 匹配模式：`[0-9]+万`、`[0-9]+千`、`[0-9]+岁`、`[0-9]+[万块元]` 等
- 示例：`手术费<span style="color: rgba(255, 69, 0, 1)">6万</span>，后续康复还要<span style="color: rgba(255, 69, 0, 1)">3万</span>`

### 标题数字标红
- 标题中的关键数字同样标红：
```html
# <span style="color: rgba(255, 69, 0, 1)">71岁</span>大嫂把房子过户给儿子，半年后住进了养老院
```

**用户二次转换说明**：用户会自行将 MD 文件中的 `<span style="color: rgba(255, 69, 0, 1)">` 格式复制到公众号后台，因此 MD 文件中无需使用公众号不支持的 Markdown 粗体 `**`，直接用 HTML `<span>` 即可。

## 格式要点

### 标题
- 一句话，有冲突/对比/数字
- 不用书名号《》
- 不用"告诉你""揭秘"等套话

### 开头hook
- 不说"刷到这个话题"
- 不说"最近看到一个故事"
- 直接场景切入：`说实话，这事我也是最近才想明白的。邻居陈姐，57岁...`

### 故事编号
- 用 `## 01` `## 02` `## 03` 而非 `## 故事A` `## 故事B`
- 每个故事后紧跟 `---` 分割线

### 对话引用
- 用 `>` 块引用
- 不加引号包裹

### 行动建议
- 用 `## 该怎么做？` 作为小标题
- 列3-4条bullet（以 `-` 开头）

### 话题标签
- 放在文末，用 `*话题标签：#退休 #健康*` 格式
- 用 `*` 斜体包裹整行

### 互动问题
- 放在文末，用 `**加粗**`
- 争议性，能让两拨人想转发讨论

---

## 禁止事项

| 禁止 | 替代 |
|------|------|
| 开头说"网友投稿" | 直接场景切入 |
| 双层书名号《《》》 | 直接标题，不加书名号 |
| 泛泛的"你身边有没有" | 具体场景+人物+动作 |
| 干巴巴叙述无对话 | 必须有 `"对话原文"` |
| 数字模糊"几年" | 必须具体"55岁""3万块" |

| 原词 | 替换 |
|------|------|
| 拉黑 | 屏蔽 |
| 逼 | 让 |
| 滚 | 走 |
