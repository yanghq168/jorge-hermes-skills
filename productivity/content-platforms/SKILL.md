# 权权内容平台矩阵

权权名下多个内容账号的每日自动内容生成系统。

## 📋 已部署 Pipeline

| 平台 | 脚本 | 定时 | 人设 |
|------|------|------|------|
| 微信公众号「围炉家常话」 | `wechat-article-daily.py` | 每天 07:00 | 人间清醒观察家 |
| 今日头条「围炉家常话」 | `toutiao-article-daily.py` | 每天 20:30 | 人间清醒观察家（头条版） |
| 小红书旅游攻略 | `xhs-travel-daily.py` | 每天 21:30 | 旅友向导 |

## 🏗️ 通用架构

```
cronjob (定时) 
  → Python脚本 (~/.hermes/cron/scripts/)
    → HTML邮件模板 
      → SMTP发送至 569545015@qq.com
        → deliver=origin 推送到当前对话
```

所有脚本遵循同一模式：
1. 选题库随机抽取（扩充内容）
2. HTML精美邮件模板（渐变色头部、卡片布局）
3. `from config_loader import get_mail_config` 读取邮件配置
4. 纯文本+HTML双版本邮件
5. 输出摘要供确认

## 🔧 用户内容偏好（已固化）

- **HTML邮件是必须的** — 用户明确抱怨"没有UI"，所有内容平台脚本必须使用HTML模板（渐变色头部、卡片布局、标签云）

- **投稿来源** — 经常有网友投稿（@xxx 格式），故事要有具体时间、地点、数字、对话
- **用词中性化** — 避免"拉黑""逼""滚"等强硬/极端情绪词汇，改用"屏蔽""让""走"等中性表达；已落地到 wechat-article-daily.py、toutiao-article-daily.py 全部引用处

- **品牌名一致性** — 所有脚本发件人统一为「权权的HERMES」，旧品牌「权权养的虾」已废弃。检查crontab和jobs.json确保无遗留旧品牌任务。同时检查脚本目录中是否有不同文件名但相同功能的旧脚本

### 公众号CSS特殊处理

CSS中的 `{` 和 `}` 必须双写为 `{{` 和 `}}`（因为HTML模板用Python `.format()` 渲染，CSS大括号会冲突）。操作方法：
1. 在HTML字符串中找到 `<style>` 和 `</style>` 之间的CSS块
2. 将所有 `{` → `{{`，`}` → `}}`
3. 其余HTML部分（变量占位符如 `{title}`）保持单大括号

## ✏️ 新增内容 Pipeline 步骤

1. 在 `~/.hermes/cron/scripts/` 创建Python脚本
2. 随机选题库设计为 `tuple` 而非 `f-string` 避免语法错误（Python 3.12 f-string不允许反斜杠）
3. 先 `python3 -m py_compile` 验证语法
4. 手动运行测试：`python3 ~/.hermes/cron/scripts/<script>.py`
5. 用 `cronjob(action='create')` 创建定时任务，`deliver='origin'`
6. 用 skill_manage `write_file` 添加参考实现

### 常见踩坑

- **f-string反斜杠**：Python 3.12禁止在f-string内使用反斜杠，先拼接中间变量再塞入f-string
- **邮件发送失败**：先检查 `python3 ~/.hermes/cron/scripts/<script>.py` 是否能单独运行成功
- **话题标签遗漏**：小红书内容必须包含 `#话题`，检查是否有 hashtags 相关变量
- **敏感词遗漏**：所有内容平台脚本必须通过 `replace()` 或模板变量实现敏感词替换，不能只依赖提示词层面的"注意用词"。必须替换的词汇包括但不限于：
  | 原词 | 替换为 | 说明 |
  |------|--------|------|
  | 拉黑 | 屏蔽 | 极端情绪词，算法过滤风险 |
  | 逼 | 让 | 攻击性词汇 |
  | 滚 | 走 | 攻击性词汇 |
- **部署到远程服务器**：内容平台脚本若需部署到远程服务器（如腾讯云），服务器通常没有GitHub凭证。应使用"本地clone + rsync"方式，而非在服务器上直接 `git clone`
- **rsync超时**：大文件/跨区域传输时rsync可能超时，改用 `tar + split + scp` 分块传输（参见 `github-repo-management` 技能的"rsync Timeouts"章节）
